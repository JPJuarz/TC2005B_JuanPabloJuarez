Questions Lab 1

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de computadoras las cuales se comunican entre si, mientras que la World Wide Web o (www)es un sistema de información que utiliza o funciona sobre el internet para acceder a documentos y otros recursos a través de enlaces. Basicamente el internet es la infraestructura que permite la comunicación entre computadoras, mientras que la web es una aplicación que se ejecuta sobre esa infraestructura para proporcionar acceso a información y servicios.

¿Cuáles son las partes de un URL?

Un URL (Uniform Resource Locator) normalmente se compone de: Protocolo (http, https), dominio (www.ejemplo.com), ruta (/pagina), y a veces otros parametros de consulta (?id=123).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Recuperar información de un recurso específico. (No modifica datos) HEAD: Obtene solo los encabezados de una respuesta sin el cuerpo del contenido. POST: Envia datos al servidor para crear o actualizar un recurso. PUT: Actualiza o reemplaza un recurso existente en el servidor. PATCH: Aplica modificaciones parciales a un recurso existente. DELETE: Elimina un recurso específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar el metodo POST porque este metodo permite enviar datos al servidor de mejor manera (mas segura) y permite enviar mayor cantidad de datos, ademas de que no expone la información en la URL como lo hace el metodo GET.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza GET ya que el navegador solicita el contenido de la pagina al servidor.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El codigo 200 significa que la solicitud fue procesada de manera correcta sin ningun error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Si es generalmente responsabilidad del desarrollador porque un error 404 (Not Found) indica que el recurso solicitado no existe o el enlace esta mal configurado, lo cual puede corregirse ajustando rutas, enlaces o archivos.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si es responsabilidad del desarrollador corregir un error 500 (Internal Server Error) porque este error indica que el servidor encontro una condición inesperada que le impidio cumplir con la solicitud, lo cual generalmente se debe a problemas en el codigo del servidor o configuraciones incorrectas.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo HTML5 se considera depreciado o desaprobado (deprecated) cuando ya no se recomienda su uso y puede ser eliminado en futuras versiones del estandar. Algunos elementos de HTML 4 que estan desaprobados en HTML5 son font, center, big, strike, entre otros ya que se han reemplazado por CSS.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Las diferencias principales entre HTML 4 y HTML5 incluyen la introducción de nuevos elementos semánticos (como header, footer, article, section), mejor soporte para multimedia (audio y video), nuevas APIs para aplicaciones web y mayor compatibilidad con dispositivos moviles.

¿Qué componentes de estructura y estilo tiene una tabla?

La estructura de una tabla incluye elementos como table para definir la tabla, tr para las filas, th para las celdas de encabezado y td para las celdas de datos. En cuanto al estilo, se pueden aplicar estilos CSS para controlar la apariencia de la tabla, como bordes, colores, espaciado y alineación.

¿Cuáles son los principales controles de una forma HTML5?

Los principales controles de una forma HTML5 incluyen input para campos de texto, textarea para áreas de texto, select para listas desplegables, button para botones, entre otros.

Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de los sistemas de información generalmente incluye las siguientes fases: planificacion, analisis, diseño, implementación, pruebas, despliegue y mantenimiento. El ciclo de desarrollo de sistemas de información se refiere al proceso iterativo de crear y mejorar un sistema a través de estas fases, asegurando que el sistema cumpla con los requisitos del usuario y se mantenga actualizado con las necesidades que siguen cambiando con el tiempo.
