Que diferencias y semejanzas hay entre Java y JavaScript?

Java y JavaScript son dos lenguajes de programación que comparten una sintaxis muy parecida y usan de estructuras y condicionales. Aun asi, tienen varias diferencias principalmente en como es que se llegan a utilizar. Javascript se usa generalmente para crear sitios web interactivos, mientras que Java se usa para crear aplicaciones de escritorio, móviles y juegos. Java es un lenguaje de programación orientado a objetos, mientras que JavaScript es un lenguaje de programación basado en prototipos.

¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)

El objeto Date tiene varios metodos, algunos de ellos son: - getDate(): Devuelve el día del mes (1-31) de una fecha - getDay(): Devuelve el día de la semana (0-6) de una fecha - getFullYear(): Devuelve el año (con cuatro dígitos) de una - getHours(): Devuelve la hora (0-23) de una fecha - getMinutes(): Devuelve los minutos (0-59) de una fecha - Date.now(): Devuelve la fecha exacta (milisegundos)

¿Qué métodos tienen los arreglos? (Menciona al menos 5*)

Al igual que date, los arreglos tienen muchisimos métodos con algunos siendo: - push(): Agrega uno o más elementos al final de un arreglo y devuelve la nueva longitud del arreglo. - pop(): Elimina el último elemento de un arreglo y lo devuelve. Este método cambia la longitud del arreglo. - length(): Devuelve la cantidad de elementos en un arreglo. - forEach(): Ejecuta una función proporcionada una vez por cada elemento del arreglo. - shift(): Elimina el primer elemento y lo devuelve.

¿Cómo se declara una variable con alcance local dentro de una función?

Una variable local dentro de una funcion se declara usando o la palabra var o let. Estas variables solo pueden ser accedidas dentro de la función donde fueron declaradas, y no pueden ser accedidas desde fuera de esa función. (a menos que de declaren desde fuera obvio)

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Utilizar variables globales dentro de funciones puede tener varias implicaciones negativas como: - Dificultar la comprensión del código, ya que no siempre es tan claro cuales variables están siendo utilizadas. - Puede causar conflictos de nombres si se declaran variables con el mismo nombre. - Puede dificultar en cuanto a la reutilización del código, ya que las funciones dependen de las variables externas.
