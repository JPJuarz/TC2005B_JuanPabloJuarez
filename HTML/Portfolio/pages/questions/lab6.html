Questions POE

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Es una buena práctica usar JavaScript para validar los inputs antes de enviar una forma porque mejora la experiencia del usuario y la eficiencia del sistema. Usar js permite dar retroalimentación inmediata (por ejemplo, avisar si un campo esta vacío o mal escrito) sin tener que recargar la pagina, lo que hace el proceso más rápido y claro para el usuario. También ayuda a reducir errores comunes y evita mandar datos incorrectos al servidor, ahorrando tiempo y recursos.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Las validaciones con JavaScript se pueden saltar porque corren en el navegador del usuario y nosotros (el usuario) podemos modificar o desactivar el código. Por ejemplo, se puede cambiar el valor de los inputs desde las herramientas del navegador o enviar los datos directamente al servidor.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Se sigue viendo como buena practica porque aunque JavaScript no es una fuente de seguridad real, sigue siendo útil como una capa de validación inicial que mejora la experiencia del usuario y reduce errores comunes antes de enviar los datos al servidor.
